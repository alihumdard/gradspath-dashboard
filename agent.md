# Agent Execution Contract

## Primary Instruction

You must execute this project only by following the milestone documents in this repository:

- milestone_1.md
- milestone_2.md
- milestone_3.md

## Mandatory Start Sequence

1. Read milestone_1.md completely.
2. Read milestone_2.md completely.
3. Read milestone_3.md completely.
4. Build a short execution checklist from those files before coding.

Do not start coding before all three files are read.

## Mandatory Implementation Order

Implement strictly in this order:

1. Milestone 1
2. Milestone 2
3. Milestone 3

## Strict Constraints

- Do not skip milestones.
- Do not mix milestone scope.
- Do not pull milestone 2 or 3 features into milestone 1.
- Do not start milestone 3 until milestone 2 is complete and validated.
- Keep all business logic inside modules only.

## Completion Gate Per Milestone

For each milestone, complete all of the following before moving on:

1. Database migrations are complete and successful.
2. Models, controllers, services, traits, and supporting classes are implemented.
3. Routes are implemented and tested.
4. Blade views for milestone scope are complete.
5. Jobs/events/background tasks for milestone scope are wired.
6. Feature tests pass for milestone-critical flows.
7. No placeholder or partial implementation remains.

## Quality Rules

- Keep implementation simple and production-ready.
- Avoid over-engineering.
- Use clear naming and module ownership.
- Keep controllers thin and services focused.
- Enforce validation, authorization, and transactions.
- Data access policy: use 80% Eloquent and 20% DB queries for heavy/complex query paths.

## Failure Policy

If any task conflicts with milestone order or scope:

1. Stop.
2. Report the conflict.
3. Continue only after realigning with the current milestone file.

## Definition of Done

The project is done only when milestones 1, 2, and 3 are each completed in order, fully validated, and all scoped features are runnable.
